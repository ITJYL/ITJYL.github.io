# 이 블로그에 대해

개발자 브랜딩에 좋은 방법 중 하나로 **GitHub + GitHub Pages**를 선택해 이 기술 블로그를 운영하고 있습니다.

---

## 선택한 방식: GitHub Pages + Chirpy

| 장점 | 단점 |
|------|------|
| ✔ 완전 개발자 느낌 | ❌ 세팅 난이도 있음 |
| ✔ 정적 블로그 (Jekyll) — 가볍고 안정적 | ❌ 디자인 감각 조금 필요 |
| ✔ 이력서·포트폴리오와 연결하기 좋음 |  |

---

## 구성

- **환경**: macOS
- **템플릿**: [Chirpy](https://github.com/cotes2020/jekyll-theme-chirpy) (Jekyll 테마)
- **호스팅**: GitHub Pages (개인 레포지토리)
- **빌드**: Jekyll 정적 사이트 생성 후 GitHub Actions로 자동 배포

GitHub에서 Chirpy 템플릿을 가져와 개인 레포지토리에 맞게 설정해 두었고, 글은 마크다운으로 작성해 커밋·푸시하면 자동으로 반영됩니다.

---

## 왜 이 방식인가

- **기술 블로그**로 쓰기 좋고, **개발자 이미지**와 잘 맞음  
- **정적 사이트**라 속도·안정성·보안 관리가 수월함  
- **Git 이력**으로 글 버전 관리가 되고, **GitHub 프로필**과 연결해 이력서에 걸기 좋음  

세팅에 손이 조금 가지만, 한 번 만들어 두면 글쓰기에만 집중할 수 있는 구조입니다.
